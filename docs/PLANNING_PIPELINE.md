# Planning Pipeline

The common ordered stages are Run Request, profile resolution, preset
resolution, planning-profile resolution, SessionPackPlan, macro-state stage,
packet grammar, Pulse Pattern, channel grammar, motif selection, EventPlan,
hard validation, advisory comparison, and diagnostics.

Session behavior comes from profile mode assignment. No planning branch checks
for Session 1. Integer sample positions are authoritative; seconds are derived.
Plans use deterministic canonical hashes and remain non-executable for rendering.
